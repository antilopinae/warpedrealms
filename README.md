# WarpedRealms Go Rewrite

Новый перенос проекта на Go разбит на два бинаря:

- `cmd/server`: авторитарный сервер на Go с HTTP-авторизацией и WebSocket-синхронизацией.
- `cmd/client`: клиент на Go с Ebitengine, локальным предсказанием, серверной реконсиляцией и интерполяцией удалённых сущностей.

## Что уже перенесено

- Базовая авторизация `sign-up` / `sign-in`.
- Lobby и отдельные raid-сессии с authoritative simulation на сервере.
- Lobby со списком рейдов и созданием новых сессий.
- Подключение клиента по WebSocket.
- Shared simulation code между клиентом и сервером.
- Client-side prediction для локального игрока.
- Server reconciliation по `last_processed_seq`.
- Интерполяция удалённых игроков и NPC.
- Raid timer, сессионные статусы игрока, лут и назначенные выходы.
- Загрузка карты `TMX/TSX`, фоновых слоёв, тайлов и спавнов из старых ассетов.

## Как запустить сервер

```bash
go run ./cmd/server
```

Сервер по умолчанию слушает `:8080`.

## Как запустить клиент

```bash
go run ./cmd/client
```

По умолчанию клиент ходит на `http://127.0.0.1:8080`.

Дополнительно:

```bash
go run ./cmd/client -server http://127.0.0.1:18080
go run ./cmd/server -addr 127.0.0.1:18080
```

## Управление в клиенте

- `Tab`: переключение между полями логина.
- `F2`: переключение `sign-up` / `sign-in`.
- `Enter`: отправить форму.
- `R`: обновить список рейдов в lobby.
- `N`: создать новый рейд и сразу войти в него.
- `Enter` в lobby: войти в выбранный рейд.
- `A/D` или стрелки: движение.
- `Space`, `W`, `Up`: прыжок.
- `E`: подобрать лут или выйти через назначенный exit.
- `LMB`: ближняя атака.
- `Esc`: закрыть игровое соединение и вернуться на экран логина.
